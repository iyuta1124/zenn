# JavaScriptで関数を使う際に気をつけること
- JavaScriptは一つの引数だけを渡すことはできない（一つ目をnullとかにする）
- 2個あるのに１こしか定義していなかったらundifined
- 同じ関数名で重複していた場合は後に定義された方が実行される
- →こういうときは関数式を使えばよい
- default値も定義できる
- 意図的に空にしたい場合はnullを使うほうが良い

# arguments
- 関数コンテキストによって自動的に定義される
- これはオブジェクトで呼び出しときに渡された実引数が渡ってくる
- あんまり使わない
- RESTパラメータで...argsを使うことが多い

# JavaScriptにおける関数とは
- 関数は**実行可能**なオブジェクトである(実行可能以外は普通のオブジェクト一緒)
- ()を末尾につけることで関数を実行する

# コールバック関数
- 他の関数に引数として渡される関数のこと
- 関数はオブジェクトなので引数としても渡せる
```js
  function hello() {
    console.log("hello")
  }

    function fn(cd) {
    cd()
  }

  fn(hello)
  --> "hello"
```
# this
- **呼び出し元のオブジェクト**への参照を保持するキーワード
- 関数コンテキスト内で設定されるもの
- グローバルコンテキスト内で使うことが多いが関数の中で使うことのほうが多い
- 実行コンテキストによってthisの参照先は変わる(関数、グローバル、イーガル)
  →呼ばれた関数ごとにthisの参照先が変わることが多い
- オブジェクトとして実行される場合→thisは呼び出し元オブジェクト
- 関数として実行される場合→thisはグローバルオブジェクト
  
:::message
- thisで何をとっているかわからなくなったときはSourcesパネルを見るとよい
- anonymousとなっている場合はSourcesのwatchでthisと検索するとでてくる
:::

## コールバック関数とthis
- コールバック関数として与えるとthisの参照先はグローバルオブジェクトが参照される

# binsとthis
- bindでthisのとりうる参照を変更できる。
```js
  const helloTom = person.hello.bind(person)
```

- bindの第一引数のオブジェクトをthisの参照先と設定し、設定した関数を実行する
- またthisのオブジェクトだけでなく、渡される引数も固定することができる
```js
function a(name) {
  console.log('hello' + name)
}
//第一引数にthisで指定したいObject,第２匹数以降に渡したい引数を固定

const b = a.bind(null, 'Tim') 

b() --> 'hello Tim'
//'Tom'を引数に指定しても固定されているためTimが呼び出される
b('TOM') --> 'hello Tim'
```

# call,applyとthis
- bind→thisや引数の参照先を変更。使用時点で実行はしない。新しく関数を設定するまで。
- call,apply→thisや引数の参照先を変更。**同時に関数を実行する。**

- callは第二引数以降に関数の引数を定義できる
- applyは第二引数に指定できるのは配列を指定する。

# アロー関数
- 無名関数を記述しやすくした省略記法
- 無名関数だとthis,arguments,new,prototypeが使えるがアロー関数だと使えない
- →特にthisの挙動は変わるので注意が必要→グローバルコンテキストを参照するようになる。
- アロー関数を宣言した時点でのthisを参照する。(オブジェクトの中で宣言したらそのオブジェクトのthisをとる)
