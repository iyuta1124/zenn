# 関数スコープとブロックスコープ

- 関数スコープ→関数の{}の部分のスコープ
- 関数の外で呼び出すとエラーとなる
---
- ブロックスコープ→{}で囲まれたもの(ifやfor)
- →**constとletを{}内で定義**し、{}外で呼び出すとエラー
  varだとブロックスコープをとらない
- 関数宣言でもブロックスコープは無視される
- 関数宣言の部分を関数式にするとスコープ外から呼べなくなる。

# スコープと実行コンテキスト
- 実行コンテキスト→コードが実行される状況
- スコープ→実行中のコードから見える範囲のこと

# レキシカルスコープ
- 語彙の〜、辞書の〜という意味
- プログラムの文脈では**どこに何を**かけているかという意味
- コードを各場所によって参照できる変数が変わるスコープのこと
- →コードを記述した時点で決定するため[静的スコープ]ともいう。
  
  自身のスコープの外側の変数を参照できる
1. 実行中のコードからみた外部スコープのこと
2. どのようにスコープを決定するの仕様のこと

# スコープチェーン
- スコープが複数階層で繋がっている状態
```js
let a = 2;
function fn1 () {
  let b = 1;
  function fn2() {
    let c = 3;
  }
  fn2()
}
fn1()
```

- 一番内側のスコープの変数を参照する
- グローバルすコープ(window)とスクリプトスコープ(windowの内側)が存在した場合はスクリプトスコープが参照される
- 変数名をつけるときは参照、実行するよりも上の行で定義する。(ホイスティング) 

# クロージャー
- レキシカルスコープの変数を関数が使用している状態
- 内側から外側の変数を参照している
```js
function fn1 () {
  let b = 1;
  function fn2() {
    console.log(b) //レキシカルスコープの変数を使用する関数
  }
  fn2()
}
fn1()
```

- 関数の外部からアクセスできない変数
- 同的な関数生成
をするためにクロージャーを使う。

## プライベート変数の定義
- 関数の内部に変数を持ちながらも外部からはアクセスできないようにする
```js
function incrementFactory() {
  let num = 0
  function increment() {
    num =  num + 1
    console.log(num)
  }
  return increment //JSだと関数を返却できる
}

const increment = incrementFactory();
//incrementが返却されるので()をつけると関数が実行される
inncrement() //1
inncrement() //2
inncrement() //3
```

## 動的な関数の生成
```js
function addNumberFactory(num) {
  function addNumber(value) {
    return num + value
  }
  return addNumber
}

const add5 = addNumberFactory(5)
const result = add5(10)
console.log(result) //15
```

# 即時関数
- 関数定義と同時に一度だけ実行される関数
- 実行結果が呼び出し基に返却される
- IIFEというとこれのこと
```js
let result = (function(仮引数){
  return 戻り値
})(実引数)
```

→関数の実行
→グループ化
