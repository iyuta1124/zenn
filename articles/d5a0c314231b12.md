# ループ文とブロックスコープ
- ブロックスコープは1ループごとにスコープが変更される
- 各ループごとに変数が宣言される。保持されているメモリ空間は別のものとなっている。
- ブロックスコープ外で宣言された変数は同じメモリ空間をとる。

# for...inと列挙可能性
- 列挙可能プロパティに対して順不同(基本順で帰ってくるが担保されていない)で反復処理を実行する
- 列挙可能性とは→各プロパティが持っているディスクリプターに含まれるenumerableという設定値のこと
- →これがtrueだと列挙対象。
- プロトタイプチェーン内も列挙対象となる→Object.hasOwnProperty()を使用する
```js
const obj = {
  prop1: 'value1',
  prop2: 'value2',
  prop3: 'value3',
}

for(let key in obj) {
    //ここでオブジェクト自体に定義されているプロパティを列挙するようにする
  if(obj,hasOwnProperty(key)) {
    // prop1,value ....prop3,value3
    console.log(key,obj[key])
  }
}
```
- Symbolで定義したプロパティは列挙対象とならない。

# for...ofと反復可能性
- イテレーターを持つオブジェクトの反復操作を行う
- イテレーター→反復操作を行う際に使用するオブジェクト(String,Array,Map,Set,aeguments,etc...)
- for ofはこのイテレーターの挙動に依存すると覚えておく
- 5個の要素を持つ配列の4番目の要素に要素が定義されていなくてもundifinedを返す
- Objectのプロトタイプは列挙しない。enumerableをfalseにしても値は帰ってくる

# MapとSet
- データを管理するための入れ物→コレクション
## ObjectとMapの違い
| | Object | Map |
| --- | --- | --- |
| キー | 文字列 | 制約なし |
| for...in | 可能 | 不可能 |
| for...of | 不可能 | 可能 |

```js
const map = new Map()
const key2 = function() {}
map.set(key2, 'value2')
console.log(map.get(key2)) // -> value2

map.delete(key2)
console,log(map.get(key2)) // -> undifined

for(const [k, v] of map) {
  console.log(k, v)  //ループできる
}
```

- Entriesという特殊な格納場所に格納される

## ArrayとSetの違い
| | Array | Set|
| --- | --- | --- |
| 重複値 | 可能 | 不可能 |
| for...in | 可能 | 不可能 |
| for...of | 可能 | 可能 |


```js
const s = new Set()
s.add(key1)
s.add(key2)
s.add(key3)
s.delete(key3)

s.has(s.has(key3)) //値があるかないかをbooleanで返す。mapでも使える

// const arry = Array.from(s)
const arry = [...s]
```

# イテレーター
- 反復操作を行う際に使用するオブジェクトのこと

```js
const s = new Set()
function genIteretor(max = 10) {
  let i = 0
  // 以下return の部分をイテレーターと呼ぶ
  return {
    //next メソッドを必ず保持。return値として{}を持つ必要がある
    next: function() { 
      if( i > max){
        done: true,
      } else {
        return {
          done:  false, //ループを継続するかどうか
          value : i++, //返却する値
        }
      }
    }
  }
}

const it = genIterator(10)
console.log(it.next()) //valueの値をincrementする

let a = it.next()
while(!a.done) {
  console.log(a.value)
  a = it.next()
}

const obj = {
  [Symbol.iterator]: genIterator.bind(null. 100)
}

for (const i of obj) {
  console.log(i) //0-99まで
}

const s = new Set(obj) //インスタンス化の際に反復可能オブジェクトを渡してあげると初期化される
console.log(s) //[[Entries]]0:0 ,1;1,...

```

# スプレッド演算子
- 反復可能や列挙可能オブジェクトの展開を行う
- let a - [...array]
- {}や[]の中で使用
- スプレッド演算子が代入された変数は新しい配列となる(別の配列)



関数の引数で呼ばれた場合残余引数(Rest Parameterと呼ばれる)
function(...args)
- 日引数で渡された変数を配列にまとめる

→スプレッド演算子はイテレーターの操作に従う

```js
const arry1 = [1,2,3,4,5]
const arry2 = [...arry1, 6]
arry2.push(6)
console.log(arry1) //(5)[1,2,3,4,5]
console.log(arry2) //(5)[1,2,3,4,5,6,6]
console.log(arry1 === arry2) // false

function(...args) {
  let ret = o
  for(let v of args ) {
    ret += v
  }
  return ret
}
const result = sum(1,2,3,4) 
console.log(result)// 10
```
