# ブラウザとスレッド
- スレッド→連続して処理される一本の処理の流れ
- メインスレッド→JavaScriptの実行とレンダリング(画面描写処理)を行う

# 同期処理と非同期処理
- メインスレッドでコードが順番に実行される
- 一つの処理が完了するまでは次の処理には進まない
- 非同期処理は一時的にメインスレッドから切り離される
- setTImeOutとかPromiseは非同期APIのため切り離される

# タスクキューとコールスタック
- 実行待ちの非同期処理の行列→非同期処理の実行順を管理
- これを先入先出しと呼ぶ(FIFO)

イベントループ→コールスタックにタスクが積まれているかどうかを定期的に確認する

# Promise
- 非同期処理をより簡単に、可読性があがるようにかけるようにしたもの
- →コールバック関数を描き続けると可読性が低くなる
```js
 new Promise(function(resolve, reject){
   resolve()
 }).then(fucntion(data)) {
   console.log(data)
 }).catch(fucntion(data) {
   console.log(data)
 }).finaliy(function(){
   console.log("終了処理")
 })
```

finallyに引数は渡せない
resolveが呼び出されたとき→then,rejectが呼び出された時にcatchに移行
thenからcatchに移行したいときはthrow new Error()などでエラーを発生させる必要がある

# Promiseチェーン
- promiseで非同期処理を順次実行していくこと
- promiseでチェーンするときにreturnするのはPromiseのインスタンス

# Promiseの並列処理
- Promise.allを使う
- return Promise([sleep(2),sleep(3),sleep(4)])
