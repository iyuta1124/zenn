# コンストラクター関数
- 新しくオブジェクトを作成するための雛形となる関数
- new演算子を使用する
- const obj = new A();
- newで作成したオブジェクトをインスタンス
- newでコンストラクタ関数を用いてオブジェクトを作成することをインスタンス化という

オブジェクトを大量に生成したい場合に有効
```js
//コンストラクタ関数は名辞的に頭を大文字にする、
function Person(name, age) {
  this.name = name;
  this.age = age;
}

const bob = new Person('bob', 18)
const tom = new Person('tom', 20)
const san = new Person('san', 19)
```

# prototype
- オブジェクトに存在する特別なプロパティのこと
- コンストラクタ関数と合わせて使用する

```js
Person.prototype.hello = function() {
  console.log('hello' + this.name)
}

bob.hello();
//メソッドとして追加したプロパティを、prototypeオブジェクトに格納しておくと
// インスタンスごとにメソッドが実行できる。
```
## prototypeを使う理由
**メモリの効率化**
this.helloをコンストラクター関数に追加することでも同じことはできるが、
インスタンスを生成するたびにハローメソッドが作成され、インスタンスに追加される。
prototypeを使うとPersonから参照したメソッドを使用するため余分なメモリを使用しない。

インスタンス化した際にprototypeの参照が__proto__ にコピーされる

# new演算子
- コンストラクター関数からインスタンスを作成するために使用する演算子
- コンストラクタ内の記述の仕方によって少し動きが変わる。

コンストラクター関数の戻り値がオブジェクト→オブジェクトを返す
戻り値がオブジェクト以外、もしくは何も返さない場合→__proto__にprototypeをコピー
このオブジェクトをthisの参照先としてコンストラクター関数を実行

```js
funvtion F(a, b) {
  this.a = a;
  this.b = b;
  return {}

  const instance = new F(1,2)
  console.log(instance)  // {}

  //しかしreturn 1 てもしくは定義されていないと
  console.log(instance) // F{a:1 , b:2}

  //return がオブジェクト以外でF.prototype.c = function(){}が定義されていると
  // F{a:1 , b:2}の__proto__にcという関数が定義される
  //戻り値がオブジェクトだと__proto__に定義されに
}
```

# プロトタイプ継承
- 別のコンストラクター関数のプロトタイプを受け継いで、機能を流用できるようにすること
 
```js
Person.prototype,hello = functon() {
  console.log('hello')
}

function Japanese(name, age){
 Person.call(this, name, age) //これでも継承できる
}
Japanese.prototype = Object.create(Person,prototype) //これで継承できる
//Japanese.prototypeにPerson{}.__proto__のhelloが追加されている
// メソッドの上書きもできる
```
- 継承→別のコンストラクター関数を受け継ぐこと

# クラス
- コンストラクター関数をクラス表記で書けるようにしたもの

# Symbol
- ES6から導入された、プロパティの重複を避けるために、必ず一位の値を返す関数。
- 既存で動いているコードを破壊しないために、実装されたもの
新しくprototypeにメソッドを定義する時に同じ名前でも登録できるようになる
- 
```js
const s = Symbol("hello")
const s2 = Symbol("hello")
console.log(s === s2) //false

const str = new String('Tom')
String.prototype[s] = function() {
  return "hello"
}

const tom = "tom"
tom[s]() //シンボルを参照したい場合は角括弧 tom,sだとプロパティが参照される
```

# プロパティとディスクリプター

- enumerable →列挙可能性

# getter/setterとstatic
getとsetの間に何か処理を挟みたい時に使う、
nameを変更した段階で、他の値を変更したい時にも使える。
```js
class Person2 {
	constructor(name, age) {
		this._name = name;
		this._age = age;
	}

	get name() {
		return this._name;
	}

	set name(val) {
		this._name = val;
	}

	static hello() {
		console.log('hello')
	}
}
```

# チェーンメソッド
一つのインスタンスに対して複数のメソッドを実行するために使用する
returnでthisを返してあげるとつなげられる
```js
class Person {
	constructor(name, age) {
		this.name = name;
		this.age = age;
	}

	hello(person) {
		console.log(`${this.name} says hello ${person.name}`);
		return this;
	}

	introduce() {
		console.log(`Hi, I'm ${this.name}, ${this.age} years old.`);
		return this;
	}

	shakeHands(person) {
		console.log(`${this.name} shake hands with ${person.name}.`);
		return this;
	}

	bye(person) {
		console.log(`Goodbye, ${person.name}.`);
		return this;
	}
}

const bob = new Person('Bob', 23);
const tim = new Person('Tim', 33);

bob.hello(tim)
	.introduce()
	.shakeHands(tim)
	.bye(tim);
```
